# 기능 명세

## 공통

### 인증
- 이메일/비밀번호 회원가입 및 로그인
- Google OAuth 소셜 로그인
- 로그아웃

### 공통 UI
- 다크모드 토글 (CSS 변수 기반)
- 반응형 UI (QA 화면은 데스크탑 전용, 나머지 화면은 반응형)
- 로딩 스피너 (API 요청 중)
- 토스트 알림 (성공 / 에러 피드백)
- 에러 페이지 (404 / 401 / 500)

### 권한 처리
- 본인 댓글만 수정/삭제 가능
- 소속되지 않은 팀의 프로젝트 접근 불가 → 401 처리
- 없는 리소스 접근 → 404 처리

### 팀/워크스페이스
- 팀 생성
- 이메일로 팀원 초대
- 한 사람이 여러 팀에 속할 수 있음
- 프로젝트는 팀 단위로 귀속
- 팀 멤버는 해당 팀의 모든 프로젝트에 접근 가능

### 사이드바 구조
```
사이드바
├── 📋 기획자
│   ├── 프로젝트 목록
│   ├── 테스트 케이스
│   └── 댓글 목록
│
├── 🛠 개발자
│   ├── 프로젝트 관리
│   ├── 피드백 목록
│   └── MCP 설정
│
└── 👤 마이페이지
```
역할 구분 없이 누구나 모든 메뉴에 접근 가능하며, UI적으로만 구분됨.

## 기획자 기능

### 프로젝트 목록
- 소속 팀의 프로젝트 목록 조회
- 프로젝트 이름 키워드 검색
- 자주 사용하는 프로젝트 북마크
- 프로젝트 선택 후 QA 화면으로 진입

### 대시보드 (프로젝트 진입 시)
- 테스트케이스 상태 현황 파이차트 (IN_PROGRESS / DONE / FAILED 비율)
- 최근 댓글 목록 요약

### QA 화면 (iframe + Agentation)

#### 레이아웃 원칙
- **iframe이 화면 전체(100vw × 100vh)를 점유** — 탭바, 헤더 등 별도 고정 UI 없음
- 내비게이션·기타 기능은 **슬라이드 사이드바**에만 존재 (평소엔 완전히 숨김)
- 항상 표시되는 요소는 **QA 툴바(floating)** 하나뿐

#### QA 툴바 (항상 표시)
- 화면 상단에 고정된 얇은 반투명 바 (iframe 위에 오버레이)
- **햄버거 버튼**: 사이드바 열기
- **DOM 선택 모드 토글**: ON 시 버튼 강조 + 상태 텍스트 표시

#### 슬라이드 사이드바
- 햄버거 클릭 시 왼쪽에서 슬라이드인 (오버레이 방식 — iframe 크기 변경 없음)
- 사이드바 외부(백드롭) 클릭 시 닫힘
- **사이드바 내 구성**
  - 프로젝트 이름 + 대상 URL
  - 테스트 케이스 목록 (상태 표시 + 추가 버튼)
  - 구분선
  - 내비게이션 링크: 프로젝트 대시보드 / 댓글 모아보기 / 마이페이지
  - 로그아웃 버튼

#### DOM 선택 흐름
- DOM 선택 모드 ON/OFF 토글
- 활성화 시 마우스 호버 요소 하이라이트
- 요소 클릭 시 Agentation 어노테이션 UI 표시 (우리 서비스 모달 아님)

### 테스트 케이스
- 테스트 케이스 생성 (제목, 설명)
- 테스트 케이스 상태 관리: 진행중 / 완료 / 실패
- 상태 필터링 (IN_PROGRESS / DONE / FAILED)

### 댓글
- 특정 DOM 요소에 댓글 작성 (Agentation으로 요소 선택)
- 댓글은 반드시 테스트 케이스에 귀속
- 댓글 목록에서 해당 요소 위치 확인 가능
- 댓글 수정/삭제 (본인 댓글만)
- 키워드 검색
- 날짜순 / 상태순 정렬
- 페이지네이션

### 댓글 모아보기

내가 작성한 QA 댓글 전체를 한 화면에서 확인·검색하는 전용 뷰.

#### URL 계층 구조 표시
댓글은 **개발물의 페이지 URL(라우트) 기준으로 그룹핑**되어 표시됨.
각 라우트가 아코디언 형태로 펼쳐지며, 그 하위에 해당 페이지에 달린 댓글 카드가 나열됨.

```
📁 /login                        (댓글 3개)
  └─ [댓글] 로그인 버튼 글자가 너무 작음  · #login-btn  · IN_PROGRESS
  └─ [댓글] 에러 메시지 위치 조정 필요   · .error-msg  · DONE
  └─ [댓글] 비밀번호 표시 토글 없음      · input[type=password]

📁 /dashboard                    (댓글 2개)
  └─ [댓글] 차트 로딩이 느림            · #chart-wrap · FAILED
  └─ [댓글] 반응형 그리드 레이아웃 깨짐  · .grid

📁 /settings/profile             (댓글 1개)
  └─ [댓글] 저장 버튼 누르면 500 오류   · #save-btn   · IN_PROGRESS
```

#### 검색 및 필터
- **키워드 검색**: 댓글 내용 · DOM 선택자 · 태그명 대상
- **상태 필터**: IN_PROGRESS / DONE / FAILED / 전체
- **정렬**: 최신순 / 오래된순 / URL 알파벳순
- 검색 중 라우트 그룹은 매칭된 댓글만 남기고 빈 그룹은 자동 숨김

#### 댓글 카드 정보
- 댓글 내용 (truncate + 펼치기)
- DOM 선택자 + 태그명
- 작성 시각

## 개발자 기능

### 프로젝트 관리
- 프로젝트 등록 (이름, URL, 대표 이미지)
- 대표 이미지 업로드 및 미리보기 (프로젝트 목록 썸네일)
- URL은 배포된 URL 또는 ngrok 등 터널링 URL
- Agentation 연동 가이드 제공
- 프로젝트 수정/삭제

### 피드백 목록
- 기획자가 달아준 댓글 목록 확인
- 댓글이 달린 DOM 요소 경로 및 위치 확인
- 상태 필터 (IN_PROGRESS / DONE / FAILED)
- 키워드 검색
- 날짜순 정렬
- 페이지네이션
- **담당자 할당**: 팀 멤버 중 한 명을 해당 피드백 수정 담당자로 지정
- **QA 결과 내보내기** (CSV / PDF) — 프로젝트별 피드백 리포트

### 칸반 보드
- 피드백 항목을 3열 칸반으로 시각화
  - **수정 필요** (IN_PROGRESS) / **수정 완료** (DONE) / **반려** (FAILED)
- 카드 클릭으로 상태 변경
- 카드에 담당자, DOM 선택자, 댓글 내용 요약 표시
- 담당자 기준 필터링

### MCP 서버 설정
- MCP 서버 연결 설정 (API 키 발급)
- 연결된 AI 도구로 댓글 + DOM 위치 정보 자동 전달
- MCP 연결 상태 확인

## 마이페이지

- 프로필 수정 (이름)
- 소속 팀 목록 확인
- 내가 작성한 댓글 수 / 최근 활동 이력
- 계정 삭제

---

## Agentation 연동 (DOM 어노테이션 레이어)

- 외부 API(Agentation) 활용으로 별도 SDK 개발 불필요
- 개발자는 프로젝트 등록 후 Agentation 연동 가이드 페이지에서 설정 확인
- Agentation이 DOM 선택, 요소 하이라이트, 댓글 UI 레이어 담당
- 어노테이션 이벤트(selector, boundingRect, content 등)를 우리 백엔드로 전달
- X-Frame-Options 헤더 제거는 여전히 필요 (iframe 로드 조건)

## 라우팅 구조

| URL | 화면 |
|-----|------|
| `/` | 랜딩 / 로그인 |
| `/signup` | 회원가입 |
| `/dashboard` | 팀 선택 / 홈 |
| `/teams/:teamId/projects` | 프로젝트 목록 |
| `/teams/:teamId/projects/:projectId` | 프로젝트 대시보드 |
| `/teams/:teamId/projects/:projectId/qa` | QA 화면 (iframe, fullscreen) |
| `/teams/:teamId/projects/:projectId/testcases/:testCaseId` | 테스트케이스 상세 |
| `/teams/:teamId/projects/:projectId/comments` | 기획자 댓글 모아보기 (URL 계층 구조) |
| `/teams/:teamId/feedback` | 피드백 목록 |
| `/me` | 마이페이지 |
| `/404` | 404 에러 |
