## Day 1 — 백엔드 기반 구축 + 인증 API

**목적**
DB, 서버 구조, JWT 인증을 처음 세팅하는 것이라 반복 구현보다 한 번에 올바른 패턴을 잡는 게 중요했음. 특히 PostgreSQL schema DDL, JWT 미들웨어 패턴은 Agent가 일관된 코드를 빠르게 생성하는 데 강점이 있다고 판단.

**요청 및 프롬프트**

| # | 요청 내용 요약 | 실제 사용한 프롬프트 (핵심 부분) | 효과적이었던 이유 |
|---|--------------|-------------------------------|----------------|
| 1 | 전체 Day 1 구현 시작 | "응 해" (계획 확인 후 실행 승인) | 계획이 충분히 구체화된 상태에서 단순 실행 승인만으로 전체 흐름이 시작됨 |

**생성 결과 vs 실제 반영**

| 항목 | Agent 결과 | 실제 반영 | 수정 이유 |
|------|-----------|----------|----------|
| db/schema.sql | pgcrypto + 9개 테이블(bookmarks 포함) + 7개 인덱스 | 그대로 채택 | data-model.md 스펙을 충실히 반영했고, bookmarks 테이블을 자체적으로 추가한 점도 features.md 요구사항에 부합 |
| db/index.js | pg Pool 싱글턴 + query 헬퍼 | 그대로 채택 | 패턴이 간결하고 실용적 |
| middleware/auth.js | JWT Bearer 검증, req.user 주입 | 그대로 채택 | 표준적인 패턴으로 추가 수정 불필요 |
| routes/auth.js | signup/login/me, 입력 검증 포함 | 그대로 채택 | 이메일 형식 검증, 비밀번호 길이 체크, 중복 이메일 처리까지 포함되어 있어 별도 수정 불필요 |
| server.js | dotenv, 라우터 마운트, SPA 폴백, 에러 핸들러 | 그대로 채택 | SPA 폴백 패턴(`/^(?!\/api).*/`)이 정확히 적용됨 |
| CSS 변수 시스템 | `:root` + `[data-theme="dark"]` 40여 개 변수 | 그대로 채택 | 다크모드 토글 즉시 동작, 뱃지/버튼/카드 스타일 모두 포함 |
| components/ | toast.js, modal.js, spinner.js | 그대로 채택 | ES 모듈 방식으로 작성되어 재사용성이 높음 |
| 로그인/회원가입 HTML+JS | 폼 검증, API 연동, 다크모드 토글 | 그대로 채택 | 클라이언트 측 검증 + 서버 측 에러 메시지 표시까지 처리됨 |

**오류 사례**

- **상황**: `public/css/style.css` 파일이 이미 존재하는 상태에서 Write 시도
- **문제점**: Write 도구는 기존 파일을 덮어쓰기 전 Read를 먼저 해야 하는 제약이 있음
- **수정 내용**: Read 후 재시도하여 해결. 실제 코드 내용에는 영향 없음

- **상황**: PostgreSQL이 설치되어 있지 않아 `psql` 명령 실패
- **문제점**: 맥OS에 기본 PostgreSQL 미설치 상태
- **수정 내용**: `brew install postgresql@16` 으로 설치 후 `brew services start postgresql@16`, `createdb qa_platform` 순으로 진행

**소감**
백엔드 기반 구축 전체를 Agent에게 맡겼는데, 파일 구조부터 API 응답 형식(`{ data, error, message }`)까지 CLAUDE.md 코딩 규칙을 정확히 준수한 채 생성해줬음. PostgreSQL 설치 문제가 있었지만 Agent가 직접 brew 설치 명령까지 실행해서 막힘 없이 진행됨. API 3개 모두 curl로 즉시 검증 완료.
