# 기능적 요구사항

# Layer 1 — 기본 구조

## 라우팅
- 3개 이상의 주요 화면 필요
  - 예: 피드 / 검색 / 마이페이지
- 동적 라우팅 구현
  - 예: `/post/:id`
- 상황별 에러 처리
  - 404 / 401 / 500 등
- 사용자 권한 제어
  - 일반 사용자는 타인 데이터 수정 불가
  - 관리자만 전체 관리 가능

## 레이아웃 / 상태 유지
- 반응형 UI 지원
- 서버 재시작 이후에도 CRUD 데이터 유지

---

# Layer 2 — 데이터 처리

## CRUD
- 목록 조회 + 상세 페이지 연결
- 데이터 생성
  - 게시글 작성 등
- 데이터 수정 / 삭제
- 입력 유효성 검사
  - 이메일 형식
  - 필수 입력값 검사 등

## 검색 / 탐색
- 키워드 검색
- 조건 기반 필터링
- 정렬 기능
  - 최신순 / 가나다순 / 인기순 등
- 페이지네이션 또는 무한스크롤

## API / 미디어 / 내보내기
- 외부 API 활용 가능
- 사용자 활동 이력 관리
- 이미지 업로드 및 미리보기
- CSV 또는 PDF 내보내기 기능

---

# Layer 3 — 사용자 경험

## 인증 / 계정
- 로그인 / 로그아웃
- OAuth 소셜 로그인
- 사용자 프로필 / 마이페이지
- 즐겨찾기 / 북마크

## 피드백 / 인터랙션
- 로딩 상태 처리
- 에러 상태 처리
- 토스트 / 모달 등 사용자 피드백 제공

## 시각화 / 디자인
- 차트 및 데이터 시각화
- 디자인 시스템 활용
- 다크모드 지원
- 다국어(i18n) 지원
