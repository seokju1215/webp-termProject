# CLAUDE.md — webp-termProject

## 프로젝트 개요

QA 협업 플랫폼. 기획자가 iframe 안의 DOM 요소에 직접 댓글을 달고, 개발자가 MCP 서버를 통해 AI로 피드백을 자동 반영하는 서비스.

---

## 기술 스택 (고정)

- **프론트엔드**: Vanilla HTML / CSS / JavaScript (프레임워크 금지)
- **백엔드**: Node.js + Express
- **DB**: PostgreSQL
- **패키지 매니저**: npm

**절대 사용 금지:**
- Firebase (DB / Auth / Storage 모두 금지)
- React, Vue, Next.js 등 프론트 프레임워크
- TypeScript
- Tailwind CSS

## 코딩 규칙

- 파일 확장자: `.html` / `.css` / `.js` 만 사용 (`.ts`, `.tsx` 금지)
- CSS: 커스텀 프로퍼티(변수)로 다크모드 구현 (`--color-bg` 등)
- JS: ES모듈 (`import` / `export`) 또는 CommonJS 중 하나로 통일
- API 응답 형식: `{ data, error, message }` 구조로 통일
- 비밀번호: bcrypt로 해시 저장
- 세션/인증: express-session 또는 JWT 사용

## Agent Log 작성 규칙

기능 구현이 끝날 때마다 `plan/agent-log-w{주차}.md`에 아래 형식으로 기록한다.
모든 상호작용을 기록할 필요는 없고, **의미 있는 상호작용**만 선별한다.

### 작성 원칙

- 프롬프트는 실제로 사용한 내용을 그대로 옮기되, 맥락을 함께 기술한다.
- Agent 결과를 그대로 쓴 경우도 **왜 그대로 썼는지** 이유를 남긴다.
- Agent 결과를 수정한 경우 무엇을 어떻게 고쳤는지 구체적으로 기록한다.
- 오류 사례가 없으면 해당 항목은 생략한다.

### 작성 형식

```
## [기능명]

**목적**
이 기능에서 Agent를 투입한 이유

**요청 및 프롬프트**

| # | 요청 내용 요약 | 실제 사용한 프롬프트 (핵심 부분) | 효과적이었던 이유 |
|---|--------------|-------------------------------|----------------|
| 1 | | | |

**생성 결과 vs 실제 반영**

| 항목 | Agent 결과 | 실제 반영 | 수정 이유 |
|------|-----------|----------|----------|
|  |  |  |  |

**오류 사례** *(해당 없으면 생략)*

- **상황**: 어떤 요청을 했는가
- **문제점**: Agent 결과의 무엇이 문제였는가
- **수정 내용**: 어떻게 직접 고쳤는가

**소감**
이 기능에서 Agent 활용이 어땠는지 한두 줄 자유롭게.
```
